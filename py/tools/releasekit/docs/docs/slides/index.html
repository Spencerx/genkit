Why Is This Hard?

1️⃣ Correct ordering — genkit (core) must publish before any plugin that depends on it

2️⃣ Ephemeral version pinning — workspace deps (genkit = { workspace = true }) must be rewritten to concrete versions for PyPI

3️⃣ Transitive propagation — if genkit bumps 0.5 → 0.6, all 61 dependents must also bump

4️⃣ Crash safety — if it fails at package #37, resume from there — not from scratch

Publish Pipeline: 6 Stages

Each package goes through 6 stages with automatic retry

1. Pin Deps
→
2. Build
→
3. Upload
→
4. Poll
→
5. Verify
→
6. Restore

Pin — Rewrite workspace = true to concrete versions

Build — uv build / pnpm pack / cargo package / dart pub / mvn package / bazel build

Upload — Upload to registry with retry + exponential backoff

Poll — Wait for registry to index the new version

Verify — SHA-256 checksum match against registry

Restore — Restore pyproject.toml to workspace form

How It Works: Genkit Integration

genkit + genkit-plugin-ollama + genkit-plugin-google-genai are core dependencies — AI is on by default 

from genkit import Genkit
from genkit.plugins.ollama import Ollama
from genkit.plugins.google_genai import GoogleGenai

ai = Genkit(plugins=[Ollama(), GoogleGenai()])

class ReleaseSummary(BaseModel):
    overview: str                    # 2-3 sentence overview
    highlights: list[str]            # Top user-facing highlights
    breaking_changes: list[str]      # Breaking changes + migration notes
    package_summaries: dict[str, str]  # Per-package one-liner

response = await ai.generate(
    model='ollama/gemma3:4b',
    prompt=f"Summarize this changelog:\n{changelog}",
    output=Output(schema=ReleaseSummary),
    config={'temperature': 0.2},
)
summary: ReleaseSummary = response.output

AI Configuration

CI-friendly defaults, cloud models available, disable with one flag

Config

# releasekit.toml
[ai]
enabled = true          # default
models  = [             # fallback chain
  "ollama/gemma3:4b",   # try first (local)
  "ollama/gemma3:1b",   # smaller fallback
  "google-genai/gemini-3.0-flash-preview",
]
temperature = 0.2
codename_theme = "mountains"
blocklist_file = ""  # extend built-in

[ai.features]
summarize       = true
codename        = true
enhance         = true
detect_breaking = true
migration_guide = true

CLI

# AI on by default
releasekit prepare

# Disable AI for this run
releasekit prepare --no-ai

# Use cloud model
releasekit prepare \
  --model google-genai/gemini-3.0-flash-preview

# Custom codename theme
releasekit prepare \
  --codename-theme animals

Fallback chain: Ollama models tried first, then Google GenAI. If all fail, falls back to non-AI behavior. Never blocks a release.
