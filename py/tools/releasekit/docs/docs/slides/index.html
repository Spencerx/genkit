ReleaseKit

Release Orchestration for Monorepo Workspaces

Publish packages in topological order with dependency-triggered scheduling,
crash-safe resume, SLSA provenance, and one-click rollback.

How do you publish 62 interdependent packages

Why Is This Hard?

1️⃣ Correct ordering — genkit (core) must publish before any plugin that depends on it

2️⃣ Ephemeral version pinning — workspace deps (genkit = { workspace = true }) must be rewritten to concrete versions for PyPI

3️⃣ Transitive propagation — if genkit bumps 0.5 → 0.6, all 61 dependents must also bump

4️⃣ Crash safety — if it fails at package #37, resume from there — not from scratch

Publish Pipeline: 6 Stages

Each package goes through 6 stages with automatic retry

1. Pin Deps
→
2. Build
→
3. Upload
→
4. Poll
→
5. Verify
→
6. Restore

Pin — Rewrite workspace = true to concrete versions

Build — uv build / pnpm pack / cargo build --release

Upload — Upload to registry with retry + exponential backoff

Poll — Wait for registry to index the new version

Verify — SHA-256 checksum match against registry

Restore — Restore pyproject.toml to workspace form

Crash Safety

CI Integration

Two-level integration: PR checks + publish workflow

PR Checks

# .github/workflows/lint.yml
on: pull_request
jobs:
  checks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: releasekit check
      - run: releasekit compliance

Publish

# .github/workflows/publish.yml
on:
  push:
    tags: ['v*']
jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      id-token: write
    steps:
      - uses: actions/checkout@v4
      - run: releasekit publish

Architecture

Pluggable backend protocols — add new ecosystems without touching core

src/releasekit/
├── cli.py              # CLI entry point
├── config.py           # releasekit.toml parser
├── graph.py            # DAG + topological sort
├── scheduler.py        # Dependency-triggered async scheduler
├── versioning.py       # Conventional Commits → semver
├── publisher.py        # Publish pipeline orchestration
├── compliance.py       # OSPS Baseline compliance
├── provenance.py       # SLSA provenance generation
├── signing.py          # Sigstore keyless signing
├── backends/
│   ├── forge/          # GitHub, GitLab, Bitbucket
│   ├── pm/             # uv, pnpm
│   ├── registry/       # PyPI, npm
│   ├── vcs/            # Git, Mercurial
│   ├── workspace/      # uv, pnpm, Go, Cargo, Dart, Maven
│   └── validation/     # OIDC, schema, SBOM, SLSA
└── formatters/         # ASCII, Mermaid, DOT, D2, ...

Quick Start

# Install
$ uv tool install releasekit

# Initialize config
$ releasekit init

# Discover workspace packages
$ releasekit discover

# View the dependency graph
$ releasekit graph --format ascii

# Run health checks
$ releasekit check

# Preview what a release would look like
$ releasekit plan

# Publish all changed packages
$ releasekit publish
